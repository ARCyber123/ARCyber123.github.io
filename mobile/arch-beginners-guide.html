Beginner's Guide to Arch Linux and its Derivatives

This is a guide for someone who has not used Arch Linux before and teaches the must-knows of Arch Linux and Arch-based distributions. I use Manjaro, which is based on Arch.

Pacman

Pacman is the default command-line package manager for Arch and its derivatives. Here are the common Pacman commands (make sure you match lowercase and uppercase, and replace package-name with the actual name of the package!):

Install a package or packages by name:

"sudo" or superuser privileges are required to install, remove or upgrade packages.
sudo pacman -S package-name
After typing this, type in your password for the superuser privileges and press enter. You will be asked for a confirmation, so type "y" for yes and press enter to proceed. To cancel, type "n" for no and press enter.

Remove a package or packages by name:

sudo pacman -Rs package-name
Just like while installing, you will be asked for the password and confirmation.

Searching for packages

Searching is essential if you don't know the name of the package. Superuser privileges are not required, since you are not making any changes to your computer. This searches the name and description of the packages.
pacman -Ss search-query
The output of this command will be:
repository-name/package-name package-version
Description of package.

Upgrading packages and system:

The same command is enough to check for updates, download package updates, install package updates and upgrade the system:
sudo pacman -Syu

Managing packages the graphical way

Ubuntu and Fedora have "rpm" or "deb" files to install packages. Arch, on the other hand, is a little different. Everything is in one of its repositories. If you don't find what you're looking for in the repositories, it will likely be in the Arch User Repository .

Building packages from Tarballs

You can also build packages by downloading a tar.gz file from the Internet. Once it is downloaded, extract the archive. In case you didn't know, these files are compressed archives just like .zip files. Open the extracted folder. You will see a file called "PKGBUILD". If you don't, you probably downloaded the source code of the package instead. Now, right click in the empty part of the folder, and click "Open in terminal". If you're using KDE Dolphin, click the menu button on the top-right of the window and click "Open terminal". You can also do this with the cd command - type in the terminal: cd /PATH/TO/FOLDER (don't forget to replace it with the location of the folder, example /home/user/Downloads/package-folder ). Now type the command: makepkg -si and your package will install.
